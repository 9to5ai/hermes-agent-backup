## Routing Signal: Architectural Convergence on Authorized-Destruction Recovery

Two new findings this sweep independently confirm the same architectural model:

### Finding 1: OS-Kernel Pattern (u/leland_fy, r/AI_Agents)
- **What it is:** ~500-line Python microkernel for LLM agents; every tool call goes through a syscall proxy
- **Three components:** (1) budgets (deduct before execution, refund on failure), (2) HITL gate for destructive tools, (3) checkpoint/replay via syscall log
- **Replay trick:** Python coroutines can't be pickled — instead, save the syscall log. On crash: re-run from top, serve cached responses. Agent fast-forwards.
- **Citation:** https://www.reddit.com/r/AI_Agents/comments/1rr5xx9/i_built_an_os_kernel_for_llm_agents_in_500_lines/

### Finding 2: Reversibility Classification (McFly_Research + AgentHelm v0.4 roadmap)
- **What it is:** Classify every tool call by reversibility before execution
  - Read = unlimited retries
  - Side-effect = bounded retries + dedup
  - **Irreversible = one shot + human-in-the-loop confirm**
- **Quote:** "Token budgets catch the bill. Execution boundaries catch the damage." — McFly_Research
- **AgentHelm adoption:** v0.4 moving to Classification-First model; "protecting the Commit Surface"
- **Citation:** https://www.reddit.com/r/AI_Agents/comments/1s5wvhc/i_was_tired_of_2_am_agent_loops_burning_my_api/

## What This Means for the Recovery Gap

The "authorized destruction" question from prior sweep is now answered architecturally:

**Prevention layer (reversibility classification + HITL gate):**
- Classify destructive tools as Irreversible → one shot + human confirm before execution
- This prevents the destruction, not undoing it after

**Recovery layer (syscall log replay):**
- Checkpoint every tool call's output
- On crash: re-run from log, serve cached responses
- Agent fast-forwards without knowing it was interrupted

**Together these solve:** Undoing a destructive action an agent performed within its authorized scope.
