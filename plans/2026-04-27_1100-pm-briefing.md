# Polymarket Daily Briefing — Plan

## Goal
Build a **daily scheduled briefing** that:
1. Fetches top trending Polymarket markets (politics, finance, geopolitics, tech, economy, election, crypto, breaking)
2. Performs deep AI research on each market
3. Delivers a **Telegram digest** + **web dashboard** with full fundamental analysis + trade recommendation

---

## Architecture

```
Daily Cron (8am UTC)
    │
    ├─► Fetch trending markets (Polymarket Gamma API, filtered by categories)
    │
    ├─► For each top market:
    │       ├─ Price history (CLOB API)
    │       ├─ Orderbook + spread (CLOB API)
    │       ├─ Recent whale trades (Data API)
    │       └─ Web research: news, sentiment, keybull/keybear
    │
    ├─► AI synthesizes → structured briefing (per market)
    │
    ├─► Save JSON → ~/projects/polybrief/data/briefing.json
    │
    ├─► Generate HTML dashboard → ~/projects/polybrief/dashboard.html
    │
    └─► Send Telegram digest → Jun (Home channel)
            └─ Includes: market summary + link to dashboard
```

**Delivery:** Telegram digest (daily) + Web dashboard (anytime)

---

## Output Format (per market)

Each market in the briefing includes:

| Field | Description |
|---|---|
| **Market** | The yes/no question |
| **Price** | Current probability (Yes %) |
| **Volume / Liquidity** | Market depth |
| **Trajectory** | Price movement over 7d/30d |
| **Whale Watch** | Notable large trades / positions |
| **📖 Background** | What the event is, why it matters |
| **🟢 Bull Case** | Reasons YES is likely |
| **🔴 Bear Case** | Reasons NO is likely |
| **📰 Sentiment** | News + social media signal |
| **🎯 Recommendation** | YES / NO / PASS with confidence (1-5★) |
| **💡 Key Takeaway** | One-paragraph synthesis for learning |

Fetch top 5 markets overall by volume, filtered if possible by these tags. If tag filtering is too narrow, take top 5 by volume and filter post-fetch.

## Cron Job Design

**Schedule:** `0 8 * * *` (8:00 AM UTC daily)

**Prompt for the cron agent (self-contained):**
1. Fetch top 5 trending markets from Polymarket Gamma API
2. For each market, get full details, orderbook, recent trades, price history
3. Run web search research for each market
4. Synthesize into structured JSON
5. Save to `~/projects/polybrief/data/briefing.json`
6. Run `generate_dashboard.py` to produce `dashboard.html`
7. Send Telegram digest to Jun using the HTML/digest content

**Skills to attach:** `polymarket`, `google-workspace` (for Gmail/Telegram)

## Risks & Tradeoffs

- **Risk:** Polymarket tag filtering may not map exactly to our categories → fallback to volume-filtered top markets
- **Risk:** AI research per market is slow (~30-60s × 5 markets = 3-5 min) → acceptable for a cron job
- **Risk:** Vercel static HTML needs redeployment → generate on-the-fly via API route instead
- **Decision:** Dashboard served from `api/briefing/route.ts` which reads the JSON and renders HTML server-side (SSR)
